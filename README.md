## Задание №1: Стейкхолдеры проекта

В таблице ниже представлен анализ заинтересованных сторон, их интересов, целей, уровня влияния на проект и потенциальных рисков.

| Стейкхолдер | Интерес к системе | Цели | Влияние | Риски |
| :--- | :--- | :--- | :--- | :--- |
| **Инвестор / Владелец** | Повышение прибыли и прозрачности бизнеса. | Увеличение загрузки, снижение расходов, достоверная отчетность. | **Высокое** (финансирование) | Потеря интереса, требование избыточного функционала. |
| **Управляющий** | Инструмент контроля и аналитики. | Контроль персонала, анализ загрузки и рентабельности, работа с жалобами. | **Очень высокое** (постановка задач) | Саботаж из-за неудобства, завышенные ожидания от MVP. |
| **Бухгалтерия** | Точность фин. учета, автоматизация расчетов. | Сверка оплат, формирование документов, контроль долгов. | **Среднее** (требования к точности) | Ошибки в данных, сложность интеграции с 1С. |
| **Менеджер по бронированию** | Удобный инструмент для быстрой обработки заявок. | Учет скидок, работа с организациями, прозрачный календарь. | **Высокое** (активный пользователь) | Ошибки в логике ценообразования, сбои в «горячий сезон». |
| **Администратор (ресепшн)** | Интуитивный интерфейс для быстрого заселения. | Быстрый поиск броней, печать документов, учет услуг. | **Высокое** (ключевой пользователь) | Отказ от работы, если система тормозит процессы. |
| **Горничная** | Простота отметки статуса номера. | Обновление статусов, получение заданий на уборку. | **Низкое** | Нежелание использовать, несвоевременный ввод данных. |
| **Клиенты (гости)** | Прозрачность, скорость, отсутствие ошибок в счетах. | Быстрое заселение, решение проблем. | **Косвенное** | Негативный опыт из-за ошибок системы. |
| **Организации-партнеры** | Удобство и прозрачность отчетности. | Учет скидок по договору, своевременные закрывающие документы. | **Низкое** | Неудобный кабинет, ошибки в спец. тарифах. |
| **IT-специалист / Админ** | Стабильность, безопасность, простота поддержки. | Наличие документации, возможность бэкапов. | **Среднее** (тех. поддержка) | Высокая нагрузка, уязвимости, отсутствие документации. |

## Задание №2: Границы системы

### Что входит в систему (In Scope)

1.  **Управление справочниками:** Корпуса, номера, услуги, организации-партнеры.
2.  **CRM:** Регистрация клиентов, история проживаний и жалоб.
3.  **Бронирование:** Создание, изменение, отмена брони, проверка доступности, расчет стоимости (скидки, тарифы).
4.  **Ресепшн (Front Desk):** Заселение/выселение, учет движения гостей.
5.  **Доп. услуги:** Фиксация оказания, автоматическое списание на счет.
6.  **Финансы:** Формирование счетов, фиксация оплат, история платежей.
7.  **Качество:** Регистрация жалоб, назначение ответственных.
8.  **Хаус-кипинг:** Управление статусами номеров, задания для горничных.
9.  **Отчетность:** Формирование отчетов по заданию (списки, свободные номера, статистика жалоб, рентабельность, история гостей).

### Что НЕ входит в систему (Out of Scope)

1.  Полноценный бухгалтерский учет (замена 1С).
2.  Email-маркетинг и сложные рекламные интеграции.
3.  Прямая интеграция с Booking.com и другими OTA (Channel Manager).
4.  Управление персоналом (HRM: графики, зарплата).
5.  Складской учет (WMS) продуктов и хозтоваров.
6.  Мобильное приложение для гостей.
7.  Управление рестораном (POS-система).

### Взаимодействие с внешними системами

На данном этапе система взаимодействует со следующим окружением:
- **Платежные шлюзы** (для онлайн-оплат)
- **Бухгалтерская система (1С)** (для экспорта данных)
- **СМС / E-mail шлюзы** (для уведомлений)
- **Госуслуги / УВМ МВД** (для передачи данных о регистрации иностранцев)

Диаграмма представляет систему как единый процесс и показывает информационные потоки между ней и внешними сущностями.

### Описание потоков данных

*Ниже представлено текстовое описание ключевых потоков.*

**Внешние сущности (Акторы):**
`Гость`, `Менеджер по бронированию`, `Администратор`, `Горничная`, `Бухгалтер`, `Управляющий`, `Организация-партнер`, `Внешняя платежная система`, `СМС-шлюз`.

**Основные входящие потоки (в ИС):**

- Подтверждения оплат (от *Платежной системы*)

**Основные исходящие потоки (из ИС):**
- Подтверждения и счета (к *Гостю*)
- Статусы номеров, подтверждения броней (к *Менеджеру*)
- Карты гостя, информация (к *Администратору*)
- Списки номеров на уборку (к *Горничной*)
- Отчеты и дашборды (к *Бухгалтеру*, *Управляющему*)
- Счета и акты (к *Организации-партнеру*)
- Запросы на авторизацию платежа (к *Платежной системе*)
