## Задание №1: Стейкхолдеры проекта

| Стейкхолдер | Интерес к системе | Цели | Влияние | Риски |
|:------------|:------------------|:-----|:--------|:------|
| **Инвестор / Владелец** | Повышение прибыли и прозрачности бизнеса | Увеличение загрузки, снижение расходов, достоверная отчетность | Высокое (финансирование) | Потеря интереса, требование избыточного функционала |
| **Управляющий** | Инструмент контроля и аналитики | Контроль персонала, анализ загрузки и рентабельности, работа с жалобами | Очень высокое (постановка задач) | Саботаж из-за неудобства, завышенные ожидания от MVP |
| **Бухгалтерия** | Точность фин. учета, автоматизация расчетов | Сверка оплат, формирование документов, контроль долгов | Среднее (требования к точности) | Ошибки в данных, сложность интеграции с 1С |
| **Менеджер по бронированию** | Удобный инструмент для быстрой обработки заявок | Учет скидок, работа с организациями, прозрачный календарь | Высокое (активный пользователь) | Ошибки в логике ценообразования, сбои в «горячий сезон» |
| **Администратор (ресепшн)** | Интуитивно понятный интерфейс для быстрого заселения | Быстрый поиск броней, печать документов, учет услуг | Высокое (ключевой пользователь) | Отказ от работы, если система тормозит процессы |
| **Горничная** | Простота отметки статуса номера | Обновление статусов, получение заданий на уборку | Низкое | Нежелание использовать, несвоевременный ввод данных |
| **Клиенты (гости)** | Прозрачность, скорость, отсутствие ошибок в счетах | Быстрое заселение, решение проблем | Косвенное | Негативный опыт из-за ошибок системы |
| **Организации-партнеры** | Удобство и прозрачность отчетности | Учет скидок по договору, своевременные закрывающие документы | Низкое | Неудобный кабинет, ошибки в спец. тарифах |
| **IT-специалист** | Стабильность, безопасность, простота поддержки | Наличие документации, возможность бэкапов | Среднее (тех. поддержка) | Высокая нагрузка, уязвимости, отсутствие документации |

## Задание №2: Границы системы

### Что входит в систему (In Scope)

- Управление справочниками (корпуса, номера, услуги, организации-партнеры)
- CRM: регистрация клиентов, история проживаний и жалоб
- Бронирование: создание, изменение, отмена брони, проверка доступности, расчет стоимости
- Ресепшн (Front Desk): заселение/выселение, учет движения гостей
- Дополнительные услуги: фиксация оказания, автоматическое списание на счет
- Финансы: формирование счетов, фиксация оплат, история платежей
- Качество: регистрация жалоб, назначение ответственных
- Хаус-кипинг: управление статусами номеров, задания для горничных
- Отчетность: формирование отчетов по заданию

### Что НЕ входит в систему (Out of Scope)

- Полноценный бухгалтерский учет (замена 1С)
- Email-маркетинг и рекламные интеграции
- Прямая интеграция с Booking.com и другими OTA (Channel Manager)
- Управление персоналом (HRM: графики, зарплата)
- Складской учет (WMS) продуктов и хозтоваров
- Мобильное приложение для гостей
- Управление рестораном (POS-система)

### Взаимодействие с внешними системами

- Платежные шлюзы (для онлайн-оплат)
- Бухгалтерская система (1С) (для экспорта данных)
- СМС / E-mail шлюзы (для уведомлений)
- Госуслуги / УВМ МВД (для передачи данных о регистрации иностранцев)

## Задание №3: Основные бизнес-процессы

| Процесс | Участники | Входные данные | Выходные данные | Описание |
| :--- | :--- | :--- | :--- | :--- |
| **Бронирование номера** | Менеджер по бронированию, клиент | Заявка клиента, даты заезда/выезда, категория номера, контактные данные | Подтвержденное бронирование, счет, гарантийный ваучер | Проверка доступности номеров на указанные даты, расчет стоимости проживания с учетом тарифов и скидок, создание брони в системе управления отелем (PMS), отправка подтверждения клиенту. |
| **Заселение (Check-in)** | Администратор (ресепшн), клиент | Паспортные данные, номер брони (или ваучер), банковская карта/наличные | Заполненная регистрационная карта (анкета), ключ-карта от номера, фискальный чек (при предоплате) | Идентификация гостя в системе, поиск или создание брони, проверка готовности номера, оформление регистрации, взятие депозита, выдача ключей и активация доступа в номер. |
| **Выселение (Check-out)** | Администратор (ресепшн), клиент | Номер комнаты, дополнительные расходы (мини-бар, услуги) | Итоговый счет, фискальный чек, статус номера "грязно" в системе | Проверка лицевого счета гостя на предмет дополнительных расходов, окончательный расчет, закрытие счета, возврат депозита, освобождение номера и смена его статуса в системе для последующей уборки. |
| **Управление уборкой** | Горничная, менеджер хаус-кипинга | Статус номера ("грязный" / "чистый"), список выехавших гостей | Сформированное задание на уборку, отметка о выполнении, обновленный статус номера | Автоматическое формирование списка задач для горничных на основе выездов гостей и запросов с ресепшн, контроль качества уборки старшей горничной, обновление статуса номера на "чистый" и готовый к заезду. |
| **Работа с жалобами** | Клиент, администратор, управляющий | Описание проблемы, номер комнаты, контакты клиента | Зарегистрированная жалоба в журнале, назначенный ответственный, отчет об инциденте | Фиксация инцидента со слов гостя, классификация проблемы (техническая, сервисная), назначение ответственного сотрудника (управляющий, техслужба), контроль устранения проблемы, уведомление управляющего о критических ситуациях. |
